Hosting a website with a custom domain in an S3 bucket

AWS's S3 storage option offers a cost effective and straightforward solution to hosting a digital portfolio or CV. I developed a Bootstrap site, used git during development for version control, and uploaded the master branch to an S3 bucket for hosting. Refer to my GitHub repository for this site here.

The basic steps in hosting a website with a custom domain in an S3 bucket are the following:

  • Register a domain for your site.
  • Create an S3 bucket for your HTML files and assets.
  • Configure your bucket to allow for public access to the Internet.
  • Route DNS traffic for your domain to your website bucket.

You can visit the official AWS documentation for completing these steps here: https://docs.aws.amazon.com/Route53/latest/DeveloperGuide/getting-started-s3.html

Take note! I opened an issue on AWS's GitHub page for these steps: https://github.com/awsdocs/amazon-route53-docs/issues/30

The documentation seems to be out of step with some of the dialogs. Read this issue to see some potential pitfalls with the documentation at present.
